## MERN Stack Developer - Junior

### ❌ Before (Generic)

```
1. Explain the difference between var, let, and const in JavaScript
2. What is the purpose of useEffect hook in React?
3. How does MongoDB store data compared to relational databases?
4. Write a function to find the largest number in an array
5. What is middleware in Express.js and how does it work?
```

### ✅ After (Realistic Interview)

```
1. Tell me about a project where you built a React component.
   How did you structure it and what challenges did you face?

2. Explain the difference between state and props in React.
   Can you give a real example from your projects?

3. What's your experience with Node.js backend development?
   Walk me through how you built an API endpoint.

4. How do you handle asynchronous operations in JavaScript?
   Give me an example from your work.

5. Describe your experience with MongoDB.
   How would you design a data schema for a blog application?
```

## Key Differences

### Real Interview Questions...

✅ **Start with context**: "Tell me about a time..." or "Describe your experience..."  
✅ **Ask for examples**: "Walk me through..." or "Give me a real example..."  
✅ **Are open-ended**: Require 2-5 minute answers  
✅ **Assess thinking**: "How would you...?" or "What trade-offs?"  
✅ **Progress naturally**: From easier to harder  
✅ **Feel conversational**: Like a human is asking

### Generic Questions...

❌ **Are textbook definitions**: Facts that can be Googled  
❌ **Have one-word answers**: "Yes/No" or simple definitions  
❌ **Don't assess real skills**: What does memorizing matter?  
❌ **Are disconnected**: No flow between questions  
❌ **Feel robotic**: Read from a script

---

## Why This Matters

Real questions reveal:

- **Actual experience** (can they talk about real projects?)
- **Communication skill** (can they explain clearly?)
- **Problem-solving** (how do they approach challenges?)
- **Growth mindset** (do they learn from mistakes?)
- **Technical depth** (can they go into detail?)

Generic questions only reveal:

- Can they memorize definitions? (Not useful)
- Do they know basic facts? (Can Google it)
- Nothing about practical skills or experience

## Real Interview Patterns

Your interviewer now uses these natural patterns:

### Opening

```
"Tell me about your experience with [technology]"
"Describe a project where you..."
"Walk me through how you've handled..."
```

### Deepening

```
"Why did you choose that approach?"
"What trade-offs did you consider?"
"How would you handle [edge case]?"
"What would you do differently today?"
```

### Probing

```
"Tell me more about that"
"Can you give a specific example?"
"How did that impact the project?"
"What did you learn from that?"
```

### Wrapping Up

```
"Any questions for me?"
"Is there anything else you'd like to add?"
"How do you feel about your response?"
```
